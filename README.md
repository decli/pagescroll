# PageScroll

一个轻量级 Tampermonkey / Violentmonkey 用户脚本：在任意网页显示一个紧凑的可拖动悬浮控件，点击后快速滚动到页面顶部或底部。

这个脚本的重点是兼容现代 SPA 页面和自定义滚动容器。它会使用 Shadow DOM 隔离样式，并在页面重渲染时自动恢复控件，尽量避免被页面样式或前端框架移除。

## 功能

- 纤细的竖向胶囊控件（约 34×72px）：`▲` / `▼` 按滚动方向上下排列，中间用细分隔线区分，方向语义一目了然。
- 默认收起为 26px 玻璃圆球，鼠标划过自动展开成完整胶囊；指针离开后展开状态保留 3 秒自动收起，悬停期间一直保持展开（拖动中、设置面板打开时不会自动收起）。触屏设备上点按圆球展开，3 秒无操作自动收起。
- 展开 / 收起是一段约 0.28s 的连续形变动画：圆球与胶囊始终对齐靠近的屏幕一侧边缘、垂直中心对齐，只向屏幕内侧生长，状态切换不会左右跳位；系统开启「减少动态效果」时自动禁用动画。
- 液态玻璃（Liquid Glass）材质：半透明磨砂、饱和度增强、边缘高光与顶部光泽，能透出页面内容的颜色；并根据控件背后内容的明暗自动在浅色 / 深色玻璃之间切换（滚动、拖动、折叠、窗口变化时实时重新采样）。
- Chromium 系浏览器（Chrome / Edge 等）额外启用透镜折射：控件边缘像凸透镜一样弯折背后的内容，并带轻微色散，观感接近 iOS 26 的液态玻璃；Safari / Firefox 及不支持 `backdrop-filter` 的浏览器逐级回退到磨砂玻璃或更实的底色，保证可读性。
- 关闭（`×`）和收起（`−`）作为次要操作默认隐藏：鼠标悬停或键盘聚焦时才从胶囊上、下边缘浮出小徽标，平时不产生视觉干扰，也不容易误触；触屏等无悬停设备上徽标常驻显示。
- 点击 `▲` 快速滚动到顶部，点击 `▼` 快速滚动到底部。
- 可拖动悬浮控件位置。
- 多滚动窗口页面中，把控件拖到哪个窗口，滚动操作就优先作用在哪个窗口。
- 拖动后在当前页面内按浏览器视口比例保持位置，窗口缩放时不乱跑。
- 默认停靠在窗口右侧、垂直 20% 高度处。
- 默认位置可配置：右键悬浮控件打开设置面板，按窗口百分比设置横/纵位置，修改即时预览（见下文）。
- 支持常规页面滚动和常见自定义滚动容器。
- 使用最高层级 `z-index`、Shadow DOM 和自动重挂载，提高在 ChatGPT 等复杂页面上的显示稳定性。

## 安装

1. 安装浏览器脚本管理器：
   - [Tampermonkey](https://www.tampermonkey.net/)
   - [Violentmonkey](https://violentmonkey.github.io/)
2. 打开脚本安装地址：
   - [PageScroll.user.js](https://raw.githubusercontent.com/decli/pagescroll/main/PageScroll.user.js)
3. 在脚本管理器弹出的安装页中点击安装。

## 使用

- 平时控件是一个小玻璃圆球；鼠标划过即展开成胶囊，移开后 3 秒自动收起，悬停期间保持展开。
- 拖动悬浮控件可以调整位置。
- 点击 `▲` 滚动到当前页面或主要滚动区域顶部。
- 点击 `▼` 滚动到当前页面或主要滚动区域底部。
- 在左右分栏等多滚动窗口页面里，把悬浮控件拖到目标窗口内，再点击上下箭头。
- 悬停时顶部边缘浮出 `×` 徽标，点击可彻底关闭当前页面上的控件（刷新后恢复）。
- 悬停时底部边缘浮出 `−` 徽标，点击可立即收起成圆球，不必等 3 秒。
- 在悬浮控件上点击右键可打开位置设置面板。

### 配置默认位置

内置默认位置在窗口右侧、垂直 20% 高度处。两种方式修改：

**方式一：设置面板（推荐）**

1. 在悬浮控件上点击右键（或点脚本管理器菜单中的「打开位置设置」）。
2. 输入横向 / 纵向位置百分比。百分比指控件中心相对浏览器窗口的位置：`0,0` 为左上角，`100,100` 为右下角，`100,20` 为右侧 20% 高度。修改会即时预览。
3. 点「保存」后在所有页面生效；点「恢复内置」清除自定义配置。

**方式二：拖动 + 菜单**

把控件拖到喜欢的位置，再点脚本管理器菜单中的「保存当前位置为默认位置」。

说明：

- 默认位置按窗口相对比例保存，不同窗口大小下自动换算。
- 配置保存在脚本管理器存储中，对所有网站全局生效；在不支持 GM 存储的环境里退回到按站点的 `localStorage`。
- 直接拖动只在当前页面生命周期内有效，刷新后回到默认位置；想长期保留就打开设置面板保存，或用菜单里的「保存当前位置为默认位置」。

## 兼容性说明

脚本会优先滚动悬浮控件中心所在或最近的滚动容器。如果页面没有匹配的内部滚动窗口，则回退到浏览器主页面滚动区域。

它对以下页面通常比普通扩展更稳：

- React / Vue / Svelte 等 SPA 页面
- 使用自定义滚动容器的页面
- 会频繁重绘 DOM 的页面
- 样式隔离较强的页面

浏览器限制导致脚本通常无法运行在以下页面：

- `chrome://` 等浏览器内置页面
- Chrome Web Store
- 浏览器扩展自身页面
- 部分浏览器内置 PDF 页面或高权限页面

## 技术实现

- `@run-at document-start`：尽早注入。
- Shadow DOM：隔离页面 CSS，避免控件被全局样式影响。
- `z-index: 2147483647`：尽量保持控件位于最上层。
- 默认位置按当前浏览器视口计算，内置为右侧 20% 高度且不遮挡滚动条。
- 自定义默认位置以控件中心相对窗口的比例存入 `GM_setValue`（无 GM 环境退回 `localStorage`）；设置面板在 Shadow DOM 内通过右键打开，`GM_registerMenuCommand` 提供菜单入口。
- 拖动位置使用当前页面内存中的视口比例记录，不持久化旧窗口尺寸下的像素坐标。
- 滚动目标优先选择悬浮控件中心所在或最近的可滚动容器，适配左右分栏滚动窗口。
- 液态玻璃自适应配色：用 `elementsFromPoint` 采样控件中心背后的元素背景色，按透明度自底向上合成有效亮度后选择浅色或深色玻璃；采样在滚动 / 拖动 / 缩放时节流触发。
- 透镜折射：用 canvas 按控件圆角形状的有向距离场（SDF）生成位移贴图，通过 SVG `feDisplacementMap` 以 `backdrop-filter: url(#…)` 应用到独立的透镜层；RGB 三通道位移强度略有差异，模拟玻璃边缘的色散。该特性仅 Chromium 系支持，其他浏览器自动跳过，不影响基础磨砂效果。
- 尺寸切换统一走「就近水平边缘锚定 + 垂直中心锚定」换算，悬停切换、保活重挂载、窗口缩放与保存的默认位置共用同一规则，杜绝收起 / 展开后的横向跳位；形变由 host 的 left/top/width/height 与面板宽高的同步 CSS 过渡完成，收起态由面板自身形变为圆球，玻璃材质全程连续。
- `MutationObserver` + 定时检查：页面移除控件后自动重挂载。
- Pointer Events：统一支持鼠标、触控板和触屏拖动。
- 主滚动容器探测：兼容 `document.scrollingElement` 和页面内部滚动容器。
